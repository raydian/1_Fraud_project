# 特征选择（信用卡欺诈检测场景）

## 一、概述

在信用卡欺诈检测中，原始交易数据往往包含大量字段，但并非每一列都对建模有正向贡献。**特征选择**旨在从候选特征中筛选出与目标相关、且有利于泛化的子集，从而提升模型效果、可解释性与运行效率。

---

## 二、为什么需要特征选择？

| 目标 | 说明 |
| ------ | ------ |
| **防止过拟合（记忆噪声）** | 若模型依赖交易唯一编号等 ID 类特征，会“记住”训练集里的个案；新交易编号未见过，预测会失效。 |
| **减少无关与虚假相关** | 持卡人姓名、卡号等与欺诈无因果关系的特征，在样本中可能偶然与标签相关，引入噪声与偏见。剔除后模型更易学到真实规律。 |
| **提升泛化能力** | 希望模型学到可迁移的欺诈模式（如“深夜大额交易风险高”），而非训练集里的个别巧合。 |
| **效率与可解释性** | 更少的特征意味着更快的训练与推理，以及更易向业务与合规解释的模型。 |

---

## 三、本数据集中的特征示例

以典型信用卡交易数据为例，常见字段包括：

| 类型 | 特征 | 说明 |
| ------ | ------ | ------ |
| 标识类 | `trans_num`, `Unnamed: 0` | 交易/行唯一编号 |
| 身份与隐私 | `cc_num`, `first`, `last`, `street`, `zip`, `dob` | 卡号、姓名、地址、出生日期 |
| 时间 | `trans_date_trans_time` | 交易发生的具体时间 |
| 交易属性 | `amt`, `category`, `merchant` | 金额、类别、商户名称 |
| 地理与人口 | `lat`, `long`, `merch_lat`, `merch_long`, `city`, `state`, `city_pop` | 经纬度、城市、州、城市人口 |
| 用户属性 | `gender`, `job` | 性别、职业 |
| 标签 | `is_fraud` | 是否欺诈（目标变量） |

---

## 四、应剔除或谨慎使用的特征

### 4.1 建议直接剔除

1. **唯一标识符**  
   `trans_num`、`Unnamed: 0` 等对每笔样本唯一。模型若依赖它们，会建立“该 ID = 欺诈”的无效映射，在新数据上无泛化价值，应删除。

2. **个人隐私与强身份信息**  
   `cc_num`、`first`、`last`、`street`、`zip`、`dob` 等。在训练集中可能因抽样偶然与欺诈相关，但：
   - 测试/线上用户多为新卡、新姓名、新地址；
   - 模型易学到“某卡号/某人易欺诈”而非通用行为模式；
   - 合规上通常禁止将此类字段用于自动化决策。  
   **结论**：不进入模型，必要时做脱敏或仅用于合规允许的审计。

3. **未加工的高精度时间戳**  
   `trans_date_trans_time` 若直接作为字符串或精确到秒的数值使用，模型可能记忆“某年某月某日某时某分某秒”与欺诈的偶然关系，无法迁移到未来时间。  
   **建议**：不直接使用，改为从中构造**泛化时间特征**（见下文“保留与构造的特征”）。

### 4.2 高基数分类特征的处理

1. **高基数类别（如 `merchant`）**  
   商户名可能有成千上万个取值，若 One-Hot 或原样编码：
   - 维度爆炸或稀疏；
   - 易记住“某商户 = 欺诈”，而该商户在测试集可能未出现。  
   **建议**：不直接入模，可考虑频率编码、目标编码（注意防止目标泄露）、或聚合为“商户风险等级”等派生特征；若无法可靠构造，可暂不使用。

---

## 五、保留与构造的特征

以下类型的特征通常**保留**或**从原始字段构造**：

| 类别 | 特征示例 | 理由简述 |
| ------ | ---------- | ---------- |
| 交易金额 | `amt` | 欺诈常与金额区间、异常大额等相关。 |
| 交易类别 | `category` | 不同类别欺诈率差异大，有统计与业务意义。 |
| 时间派生 | `hour`, `day_of_week`, `month`, `is_weekend` | 从时间戳提取，表达“时段/星期/是否周末”等可迁移模式。 |
| 地理与人口 | `lat`, `long`, `merch_lat`, `merch_long`, `city`, `state`, `city_pop` | 欺诈常有地域、商户位置与人口规模等模式。 |
| 用户属性 | `gender`, `job` | 可与风险有弱相关，在合规允许前提下可保留。 |

**原则**：保留“可迁移、可解释、与业务认知一致”的特征；时间、地域、金额、类别等通常符合这一点。

---

## 六、特征选择的常用方法（补充要点）

除基于业务与数据理解的“人工筛选”外，可结合以下方法做系统化特征选择：

| 方法类型 | 代表方法 | 简要说明 |
| ---------- | ---------- | ---------- |
| **过滤法（Filter）** | 相关系数、方差阈值、IV/WOE、卡方检验、互信息 | 按与目标的相关性/信息量或特征自身方差排序，去掉低相关或低方差特征；不依赖具体模型。 |
| **包装法（Wrapper）** | 前向/后向选择、递归特征消除（RFE） | 以模型效果（如 AUC）为准则，迭代增删特征；计算量大，适合特征数中等时使用。 |
| **嵌入法（Embedded）** | 树模型特征重要性、L1 正则（Lasso） | 在模型训练过程中得到特征权重或重要性，一次训练即可得到排序或稀疏解。 |

**实践建议**：

- 先做**过滤法**快速筛掉明显无用特征，再结合**嵌入法**看重要性，必要时用 **Wrapper** 做精细子集选择。
- 划分训练/验证/测试集后，**所有基于目标或统计量的筛选（如 IV、目标编码）都只在训练集上计算**，再应用到验证/测试集，避免数据泄露。

---

## 七、效果与原则小结

通过“去粗取精”的特征选择，模型更易学到**可泛化的欺诈规律**，例如：

- "凌晨时段 + 高金额 + 某类商户”组合风险更高；
- 而非记忆“某笔交易在 2020-06-21 12:14:25 消费 2.86 元为正常”。

**核心原则**：  
剔除唯一 ID、隐私字段和未加工的细粒度标识；保留或构造具有业务含义、可迁移的时间/金额/类别/地理等特征；并辅以过滤法、嵌入法（及必要时包装法）做系统化筛选与验证。
