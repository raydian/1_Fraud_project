# 金融交易风控自动化机器学习项目

面向金融交易风控场景，构建从**机器学习学习理解 → 自动化优化 → 实战落地 → 系统化上线 → Agent 闭环改进**的全流程 AutoML 项目体系。
强调：**先掌握必要知识内容理解**，再进入具体**操作与实践**（尤其第三、第四部分）。

---

## 第一部分：机器学习学习与理解（模型体系 + 风控评估）

> 目标：不仅“会用”，更要理解不同模型的适用条件、优势劣势、可解释性与工程约束，为后续 AutoML 的模型搜索空间打基础。

### 1.1 金融风控问题建模认知（必须理解）

- 风控本质：**不平衡二分类 + 时间序列 + 分布漂移 + 可解释合规**
- 风控数据常见结构：用户/设备/商户/地理/行为序列/历史聚合
- 样本问题：标签延迟、欺诈策略变化、黑产对抗、数据泄露风险

### 1.2 必备评估指标与风控视角（必须理解）

**离线指标（建模阶段）**

- AUC-ROC：排序能力
- KS：风控强相关指标（区分度）
- PR-AUC：不平衡更敏感
- Precision / Recall / F1：策略层面权衡
- 混淆矩阵：误杀/漏放代价分析

**线上与稳定性指标（落地阶段）**

- PSI：分布漂移（关键）
- AUC/KS 时间滚动监控
- 拒付率/投诉率/人工审核通过率：业务闭环

### 1.3 时间切分与数据泄露（风控必须掌握）

- 为什么不能随机切分：未来信息泄露
- 必须掌握：
  - TimeSeriesSplit / 时间窗口划分
  - 按用户维度的切分策略（避免同一用户穿越训练与测试）
  - 标签延迟处理与 “观察窗/表现窗” 概念

---

### 1.4 模型体系学习与理解（重点：不同模型要“理解到位”）

#### A. 逻辑回归 LR（基线 + 可解释）

**要理解：**

- 线性可分假设、概率输出与阈值策略
- 正则化（L1/L2）对稳定性与特征选择的影响
- 适用场景：强解释、特征已分箱/WOE、合规要求高
**风险点：**
- 对非线性关系弱，需要特征工程补足

#### B. 决策树 / 随机森林 RF（非线性 + 稳健）

**要理解：**

- 决策树的分裂、过拟合原因、剪枝思想
- RF 的 Bagging、方差下降机制
**适用场景：**
- 数据量中等、需要稳健但延迟允许
**风险点：**
- 线上延迟、解释性相对弱于 LR（但可用特征重要性/SHAP）

#### C. GBDT 系列：XGBoost / LightGBM / CatBoost（工业主力）

**要理解：**

- Boosting 思想：逐步拟合残差
- 树深度/叶子数/学习率的偏差-方差权衡
- 类别特征处理差异：
  - LightGBM：需要编码或原生支持（视实现）
  - CatBoost：对类别特征友好
  **适用场景：**
- 风控最常用：效果强、特征适配广
**工程约束：**
- 训练与推理延迟、特征一致性、线上可解释输出（SHAP）

#### D. SVM / 线性模型家族（选学）

**要理解：**

- 大样本下训练成本
- 核方法适用边界
**适用：**
- 小数据、特征维度高但样本不大

#### E. 神经网络 / 深度学习（可选进阶）

**要理解：**

- 为什么风控常用树模型多于深度：数据结构与解释性、上线成本
- 若使用：Embedding（类别）、序列建模（RNN/Transformer）对行为序列更强
**适用：**
- 大规模行为序列、复杂模式、强算力与工程支持

## 第二部分：特征工程基础 + 自动化特征工程建设

> 目标：先掌握特征工程“基础知识结构”，再建设“自动化特征工程系统”，为 AutoML 打底。

### 2.1 特征工程基础知识（必须理解）

**数据类型与处理**

- 数值型：缺失、异常、缩放（Standard/Robust）
- 类别型：One-Hot / Target Encoding / Frequency Encoding
- 时间型：时区统一、时间粒度、节假日/周期性
- 文本/地址等：基础清洗（可选）

**风控常用特征结构**

- 静态画像特征：用户/设备/商户
- 行为统计特征：频次、金额分布、异常比例
- 关系特征：用户-设备-商户关联网络（可选）

### 2.2 风控核心特征：滑动窗口与历史行为（必须理解）

- 观察窗（过去多久统计）与表现窗（未来多久标签）
- 常用窗口：1h / 6h / 1d / 7d / 30d
- 典型统计：count/sum/mean/max/std/unique_ratio/拒绝率

### 2.3 分箱、WOE、IV（风控专属基础）

- 分箱方法：等频/等距/卡方/最优分箱（约束单调）
- WOE：增强线性可解释模型
- IV：变量区分度与筛选依据
- 稳定性：分箱对抗漂移的价值（增强鲁棒）

---

### 2.4 自动化特征工程系统建设（从理念到模块）

**目标：让系统自动完成：处理 → 生成 → 选择 → 解释 → 版本化**

#### 模块 A：自动特征处理（Auto Preprocess）

- 类型推断（数值/类别/时间）
- 缺失处理策略自动选择
- 编码策略自动选择（低基数 One-Hot，高基数 Target/Freq）
- 缩放策略自动选择（异常多用 RobustScaler）

#### 模块 B：自动特征生成（Auto Feature Generation）

- 时间派生：小时/星期/是否周末/节假日
- 滑动窗口聚合：用户/商户/设备粒度统计
- 交叉特征：用户×商户、设备×地区（可控生成，防爆炸）

#### 模块 C：自动分箱与 WOE（Auto Binning）

- 自动分箱（带单调约束可选）
- 自动输出 WOE/IV 表
- 输出可审计的分箱规则（可落地）

#### 模块 D：自动特征选择（Auto Feature Selection）

- Filter：相关/互信息/方差阈值
- Embedded：LGBM importance、L1 正则
- 稳定性筛选：PSI/时间分片重要性一致性

#### 模块 E：特征版本与一致性（Feature Registry 思想）

- 特征定义版本化
- 训练/推理一致性检查
- 数据漂移与特征可用性监控

## 第三部分：自动优化（模型选择与 HPO）——先理解再实践

> 目标：理解 AutoML 的核心“搜索与优化”原理，然后再搭建可运行的自动训练与调参系统。

### 3.1 必要知识内容理解（先学）

#### 3.1.1 搜索空间是什么

- 模型搜索空间：LR / RF / LGBM / CatBoost …
- 超参空间：树深、学习率、正则、采样、叶子数等
- 特征空间：不同特征子集、分箱策略、编码策略

#### 3.1.2 搜索策略是什么

- Grid Search：全覆盖，成本高
- Random Search：更有效率
- 贝叶斯优化（TPE/SMBO）：在有限预算下更聪明
- Hyperband / Successive Halving：资源分配 + 早停

#### 3.1.3 风控的多目标优化（必须理解）

不能只追 AUC：

- AUC / KS（效果）
- 推理延迟（实时）
- 稳定性（PSI / 时间一致性）
- 可解释性（合规）

=> 需要**多目标或带约束的优化**。

## 第四部分：端到端 AutoML 流水线——先理解系统，再落地工程

> 目标：先理解工业级 AutoML 系统应该长什么样，再做端到端流水线工程化。

### 4.1 必要知识内容理解（先学）

#### 4.1.1 端到端流水线的核心要求

- 单入口：从原始数据到模型一键运行
- 可复现：数据版本、代码版本、配置版本
- 可回溯：每次训练可追踪原因与差异
- 可部署：模型与特征一致性、推理接口规范
- 可监控：效果、漂移、延迟、报警

#### 4.1.2 模型管理与上线流程（必须理解）

- Model Registry：版本、指标、元信息、签名
- 部署方式：批处理 / 实时服务
- 回滚机制：A/B、灰度、阈值策略调整
- 线上监控：AUC/KS、PSI、数据质量、延迟
